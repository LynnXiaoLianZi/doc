Command and FISH Highlighting in the Program

In two cases in this documentation, commands and FISH, when shown, will match the default syntax highlighting of the program (assuming the default is in use — syntax highlighting may be customized by the user in the Options dialog). These are the case of a complete data or FISH file listing, or the case where lines of commands or FISH are presented offset from the main body text, as here. In both cases, the lines are boxed as shown.

block edge apply stress -1, 0, -2 ...
                 range position-x 22.7 1.19e-1

When presented inline with body text (either plain or as a hyperlink), or in the reference sections, or elsewhere, the presentation is simplified somewhat. Command words and keywords are presented in black monospaced text; values supplied with keywords or arguments, regardless of type, are shown in blue text.

To illustrate: a reference in the body text such as this — block create — is a basic hyperlink to the reference documentation for the command. This is quite common. Similarly, an inline illustration of a “filled out” command is common, and would look something like this:   block zone generate edge-length 3.14.
