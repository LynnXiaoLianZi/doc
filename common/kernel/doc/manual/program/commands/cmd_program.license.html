program license command

Syntax

program license keyword

Primary keywords:

list    location

Get/Set license information.

Note

As with all program commands, use of the command word program is optional; program list and list are both valid.

The following keywords are available

list keyword

List license information. The following sub keywords are available.

key-information

Look for a valid Itasca security key directly connected to the computer and report all of the information on it. The key does not have to contain a valid license for this code, it may be for any Itasca code. Only one Itasca key can be directly connected to the computer for this operation.

security

Look for a valid license(s) (using the current location settings) and report the contents of the license(s) if found. The license information will be for the current license requirements, or if there are none it will use the minimum default requirements.

serial

List the serial number(s) of the licenses that meet the current license requirements. If there are no current requirments, it will use the minimum default requirements.

location keyword

Specify the location used to obtain any licenses used the satisfy the current requirements. The following license location keywords are available:

key-local <key-id i >

Specify that the license(s) can be found on SuperPro hardware keys directly connected to the computer via a USB port. The optional key-id keyword can be used to specific a specific key be used by entering a key serial number.

key-network <server > <key-id i >

Specify that the license(s) can be found on a network server that is directly connected to a SuperPro network hardware key. The server name must be indicated immediately following - this can be an ip address (as a string), the name of a computer on the local network, or a URL. If no server name is specified, then the system will use whatever server was last specified for a network key license. Use the string localhost to indicate the network key and server is the current computer. The optional key-id keyword can be used to specific a specific key be used by entering a key serial number.

requirements

Lists the license requirements for the current model. Within each group, a license which satifies any of them will fullfill the requirements. All groups must find a license which satisify one entry in each group. There may be no requirements at all.

web <sfile >

Specify that the license(s) can be found using a web license. The string following must indicate the location of a .lic license file, which will be loaded to indicate the license information used. If no file is indicated, the system will use whatever was last given as a web license identifier.
