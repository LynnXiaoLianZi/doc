model configure command

Syntax

model configure keyword ...

Configure an additional calculation mode. This command allows the user to specify that the model is using calculation modes that either require extra resources or are restricted by your license information. The model.configure command can be given at any stage of an analysis, but it must be given before the calculation mode (model solve or model cycle) can be invoked.

The following keywords apply.

array i

Set the size of UDEC’s main array memory in megabytes.

axisymmetry

Specifies that UDEC will be using an axisymmetric solution. The axis of rotation is the y axis. This mode works for mechanical calculation for deformable blocks only. It cannot be used with fluid flow, thermal calculation, or structural elements.

cell i1,i2

Alternate contact detection logic. i1, i2 refers to the number of cells in the x- and y-directions. This refers to the method used in UDEC to determine which blocks to consider for contact. This method is slower than the “domain” logic, but allows models with blocks that may bounce from one block to another. Fluid flow is not available with the cell logic.

cppudm *

C++ user-defined models and/or FISH intrinsic plugins.* FISH plugins can be loaded (but cycling is disabled) without this configuration.

creep

Creep material analysis*

energy

Configures UDEC to sum energy components whie cycling.

fluid

Fluid-flow analysis

plugin *

C++ user-defined models and/or FISH intrinsic plugins.* FISH plugins can be loaded (but cycling is disabled) without this configuration.

plane-stress

plane stress analysis (only for elastic analysis or block zone cmodel assign strain-softening)

thermal *

Configure for thermal analysis.


Note

*These analysis modes are available only if the corresponding program option is included in the program license.
