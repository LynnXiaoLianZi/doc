model cycle command

Syntax

model creep b keyword

Primary keywords:

history    list    timestep    time-total

Set parameters for a creep material analysis. Time-dependent creep material analysis is only available if model configure creep has been specified. history keyword
timestep

Store history of creep timstep.

time

Store history of creep time.

list

Display creep calculation-mode information.

timestep

Timestep controls for creep calculations. Note that there is no analytical solution for the stable timestep for most creep models, therefore the timestep in automatic mode is controlled by the unbalanced force. The following keywords are available.

automatic b

If on, the stable timestep is determined according to the timestep servo with settings specified below. If off the timestep must be set manually using the fix keyword.

fix f

Set the timestep to be the fixed value f. All timestep calculations are skipped.

latency i

Set the minimum number of steps that must elaspe before the timestep is changed when the servo is controlling the timestep. Using this keyword implicitly sets servo on. By default the :latency: is 1.

lower-bound f

If the current unbalanced force ratio falls below f, then the lower-multiplier is applied to the time step to increase it. Using this keyword implicitly sets servo on. By default the :lower-bound: is 1e-3.

lower-multiplier f

The value applied to the current timestep when the unbalanced force ratio falls below lower-bound. This value should be greater than 1.0. By default lower-multiplier is 2.0.

maximum f

The maximum timestep allowed. The servo will not raise the timestep above this value. There is no default timestep maximum.

minimum f

The minimum timestep allowed. The servo will not lower the timestep below this value. There is no default timestep minimum.

starting f

Specify the timestep to use at the start of creep calculations, when the timestep is under servo control. The timestep will be adjusted by the servo every step afterwards. A starting timestep must be given before starting creep calculations.

upper-bound f

If the current unbalanced force ratio falls above f, then the upper-multiplier is applied to the time step to increase it. Using this keyword implicitly sets servo on. By default the :upper-bound: is 5e-3.

upper-multiplier f

The value applied to the current timestep when the unbalanced force ratio falls above upper-bound. This value should be less than 1.0. By default lower-multiplier is 0.5.

time-total f

f is the accumulated creep time for the model. This can be reset to any value, and will continue to be accumulated during cycling after that.
