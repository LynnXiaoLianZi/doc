block dynamic free-field command

Syntax

block dynamic free-field <b > keyword <range>

Primary keywords:

base    change    delete    generate    history    initial    list    properties    range    reference    top    history

Manually manipulate the free-field boundaries. In most cases, it is easier to use the automatic free field (see block edge apply free-field). The dynamic free field consists of a one-dimensional finite-difference calculation executed in parallel with the main calculation, and provides the lateral boundary conditions for dynamic analysis in which a vertically propagating plane wave is applied to the base of the model. The boundary nodes to be linked to the free field should be defined by using the block edge apply free-field command. The free-field boundary may be applied to one or both lateral boundaries of the model. Both lateral boundaries should be vertical, but they may have different heights. See Section 4 in Special Features and Section 4 in the Example Applications for further explanation and application of this command.

base keyword

Apply free field boundary conditions to the base.

history <keyword>

Time varied history (see note 1).

stress-xy f

Set \(xy\)-stress.

stress-yy f

Set \(yy\)-stress.

velocity-x f

Set \(x\)-velocity.

velocity-y f

Set \(y\)-velocity.

viscosity-x f

Set the viscous boundary in the \(x\)-direction.

viscosity-y f

Set the viscous boundary in the \(y\)-direction.

change keyword
material i

Assign material number i to free-field zones.

model i

Assign constitutive model i to free-field zones.

delete

Delete the free-field boundary.

generate keyword

Generate the free-field mesh.

left

Generate only the left-hand side free-field (optional).

npoints i

Set the number of free-field gridpoints (required).

right

Generate only the right-hand side free-field (optional).

yrange f f

Set the lower and upper \(y\)-coordinates (required).

history keyword

Get a free-field history of one of the following quantities.

acceleration-x fy i

Get history of free-field \(x\)-acceleration at fy for side i (1 = left, 2 = right).

acceleration-y fy i

Get history of free-field \(y\)-acceleration at fy for side i (1 = left, 2 = right).

displacement-x fy i

Get history of free-field \(x\)-displacement at fy for side i (1 = left, 2 = right).

displacement-y fy i

Get history of free-field \(y\)-displacement at fy for side i (1 = left, 2 = right).

stress-xx fy i

Get history of free-field \(xx\)-stress at fy for side i (1 = left, 2 = right).

stress-xy fy i

Get history of free-field \(xy\)-stress at fy for side i (1 = left, 2 = right).

stress-yy fy i

Get history of free-field \(yy\)-stress at fy for side i (1 = left, 2 = right).

velocity-x fy i

Get history of free-field \(x\)-velocity at fy for side i (1 = left, 2 = right).

velocity-y fy i

Get history of free-field \(y\)-velocity at fy for side i (1 = left, 2 = right).

initial keyword

Initialize free field stresses, pore-pressures, displacements, and velocities.

displacement-x f

Set gridpoint \(x\)-displacement.

displacement-y f

Set gridpoint \(y\)-displacement.

pore-pressure f keyword

Set zone pore-pressure.

fgrad

Add gradient to value such that actual_value = value + fgrad \(\cdot y\).

stress-xx f

Set zone \(xx\)-stress.

stress-yy f

Set zone \(yy\)-stress.

stress-xy f

Set zone \(xy\)-stress.

stress-zz f

Set zone \(zz\)-stress.

velocity-x f

Set gridpoint \(x\)-velocity.

velocity-y f

Set gridpoint \(y\)-velocity.

list keyword

List free-field data. The following keywords are available.

boundary

List free-field boundary conditions.

displacement

List free-field displacements.

property

List

reference

List reference stresses.

state

List free-field general data.

stress

List free-field stresses.

properties keyword

Set far-field properties.

bulk f

Set far-field bulk modulus.*

density f

Set far-field density.*

poisson f

Set far-field Poissons ratio.*

shear f

Set far-field shear modulus.*

young f

Set far-field Youngs modulus.*

material i

Set far-field property number.

Note

*Far-field elastic properties can be specified in three ways: by material number (in which case the properties are specified using the PROPERTY command); by bulk and shear modulus; and by Youngs modulus and Poissons ratio.

range keyword

For the properties set with the property keyword, an optional far field range may be specified using the following range phrases.

left

Apply only to left-hand side free-field mesh.

right

Apply only to right-hand side free-field mesh.

yrange fyl fyu

Apply the range to the region fyu < \(y\) < fyl.

reference stress

Take current free-field stresses as static reference stresses. This command is automatically executed when the free-field is applied to the grid (i.e., block edge apply free-field).

top keyword

Apply free field boundary conditions to the top.

history <keyword>

Time varied history (see note 1).

stress-xy f

Set \(xy\)-stress.

stress-yy f

Set \(yy\)-stress.

velocity-x f

Set \(x\)-velocity.

velocity-y f

Set \(y\)-velocity.

viscosity-x f

Set the viscous boundary in the \(x\)-direction.

viscosity-y f

Set the viscous boundary in the \(y\)-direction.

Note 1

Histories - A time varying history may be applied to a free-field top or bottom.

history keywords

The keyword history defines a time history that is a multiplier for all the stresses and x- and y-velocities given on the same input line.

stresses and velocities can be applied as a constant, sinusoidal or user-defined FISH function, or they may consist of a series of points given by a TABLE command or read from a file. The keywords must be given on the same line specifying the load, stress or velocity.

cosine ffreq ftime

Get a history of cosine wave load with frequency ffreq (cycles/sec) applied for a problem time period of ftime.

@funcname

Use the user-defined fish function named funcname as the history multiplier.

sine ffreq ftime

Get a history of cosine wave load with frequency ffreq (cycles/sec) applied for a problem time period of ftime.

table i

Input the history via the trick table commands. Table i consists of a list of pairs: \(time\), \(f(time)\). Linear interpolation is performed between the given discrete points.
