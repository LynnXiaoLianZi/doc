block creep timestep command

Syntax

block creep timestep keyword <keyword>

Primary keywords:

automatic    fix    latency    lower-bound    lower-multiplier    maximum    minimum    starting    upper-bound    upper-multiplier

Sets the current creep timestep.

automatic

Set current creep timestep to be automatically adjusted.

fix f

Set the current creept timestep to the fixed value f.

latency i

Set the number of creep timesteps that the maximum out-of-balance force must be above or below the set limits (block creep timestep lower-bound or block creep timestep upper-bound) before the timestep is changed. This prevents spikes from changing the timestep (requires the automatic command). The default value is 100.

lower-bound f

Set the limit for the maximum out-of-balance force that will allow the timestep in creep calculations to be increased (requires the block creep timestep automatic command). The default value is 1e4.

lower-multiplier f

Set the multiplication factor used to increase the creep timestep if the maximum out-of-balance force drops below block creep timestep lower-bound for more than block creep timestep latency timesteps (requires the block creep timestep automatic command). The default value is 2.0.

maximum f

Set the upper limit for the creep timestep when block creep timestep automatic is used. The default value is 1e4.

minimum f

Sets the lower limit for the creep timestep when block creep timestep automatic is used. The default value is 1e2.

starting f

Specify the timestep to use at the start of creep calculations, when the timestep is under servo control. The timestep will be adjusted by the servo every step afterwards. A starting timestep must be given before starting creep calculations.

upper-bound f

Set the limit for the maximum out-of-balance force at which the timestep in creep calculations may be decreased (requires the block creep timestep automatic command). The default value is 1e5.

upper-multiplier f

Set the multiplication factor used to lower the creep timestep if the maximum out-of-balance force exceeds block creep timestep upper-bound for more than block creep timestep latency timesteps (requires the block creep timestep auto command). The default value is 0.5.
