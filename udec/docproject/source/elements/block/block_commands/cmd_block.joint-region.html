block joint-region command

Syntax

block joint-region keyword <range>

Primary keywords:

id    list

Define a joint region. A convex quadrilateral region is defined for generation of a joint set (or sets) and assigned id n. The coordinates of the four corners (in a clockwise direction) delimit the boundary of the region. The region boundary must be convex. A joint set defined by the BLOCK CUT JOINT-SET command will be generated in all blocks or portions of blocks within this region. If the region is not specified, it is assumed to be the entire problem domain. The BLOCK JOINT-REGION command can also be used to limit the action of the BLOCK CUT CRACK command.

The optional keyword delete will cause stored joints from previous BLOCK CUT JOINT-SET or BLOCK CUT CRACK commands to be deleted within the region. This is useful when generating joints in multiple JREGION regions to avoid cuts by joints generated in adjacent regions. NOTE: BLOCK JOINT-JREGION may also be used to define quadrilateral regions to be used in the range phrase of any command (e.g., … range jregion n).

id i keyword

Create or delete a joint region with the id i .

fx1 fy1 fx2 fy2 fx3 fy3 fx4 fy4

Specify the coordinates of the four corners (in a clockwise direction) to delimit the boundary of the region. The region boundary must be convex. The jregion can be specified in the range for the block cut commands.

delete

Delete stored joints from previous JSET or CRACK commands. This is useful when generating joints in multiple JREGION regions to avoid cuts by joints generated in adjacent regions.

Note

Joint regions defined with this command may also be used as keywords in a range phrase (e.g., … range joint-region i).

list

List defined joint regions.
