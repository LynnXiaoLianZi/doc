block insitu command

Syntax

block insitu keyword <range>

Primary keywords:

aperture    block-only    joint-only    nodis    pore-pressure    stress-principal    stress    stress-zz    water-table    zone-pressure    stress-topography    overburden-xx    overburden-yy    overburden-zz

Initializes block zone and contact stresses, domain pressures, zone pore pressures, and hydraulic apertures. Initial zone stresses in all deformable blocks and stresses along joints between rigid blocks and/or deformable blocks in the optional range are set. If no range is specified, stresses are initialized for the entire model. Remember that compressive stresses are negative. The following keywords apply.

aperture f

Set contact hydraulic aperture. This also sets the APERTURE-ZERO-LOAD parameter for each contact so that \(aperture = aperture-zero-load + u_n\), where \(u_n\) is the normal displacement.

block-only

Initialize stresses only in block zones (not in contacts).

joint-only

Initialize stresses only in contacts (not in block zones).

nodis

Inhibit calculation of joint normal displacements (i.e., joint initial closure).

Note

Domain pressures may be initialized with the following pore-pressure keywords. These keywords should be given by the same INSITU command that initializes stresses (see Note 1, below).

Pressures in all domains except the outer domain will be initialized to

\[p = po + x · xg + y · yg\]

where (\(x\),:math:\(y\)) are the coordinates of the domain center.

pore-pressure f <keyword>

Set fluid pressure in domains (default is \(p_o\) = 0.0).

pressure-gradient-x f

Set fluid pressure gradient in the \(x\)-direction (default is f = 0.0)

pressure-gradient-y f

Set fluid pressure gradient in the \(y\)-direction (default is f = 0.0)

stress-principal

Alternate method to define the stress state is by specifying a stress measurement at a point and an optional gradient. The use of this command requires that the global orientation of the UDEC plane be specified using the commands model orientation dip-direction and model orientation origin. The keywords are:

s1 f

major principal stress

s2 f

intermediate principal stress

s3 f

minor principal stress

dip-s1 f

major principal stress dip

dip-direction-s1 f

major principal stress direction

dip-s2 f

dip intermediate principal stress dip

dip-direction-s2 f

intermediate principal stress direction

The following are optional keywords:

gradient-vertical f

gradient in the veritcal direction

location fx fy fz

The location specifies the coordinates of the stress measurement. The gradient is the stress change in the vertical direction. All other stresses are incremented proportionately by the gradient to prevent stress rotation.

stress fxxo fxyo fyyo <nodis>

Initialize the \(xx\)-stress, \(xy\)-stress and \(yy\)-stress components to the values fxxo, fxyo, fyyo.

gradient-x fxxx fxyx fyyx

Lineary vary initial stress initialized at origin (0,0) in the \(x\)-direction (and see next equation).

gradient-y fxxy fxyy fyyy

Lineary vary initial stress initialized at origin (0,0) in the \(y\)-direction.

\[\begin{split}fxx = fxxo + (fxxx · x) + (fxxy · y) \\ fxy = fxyo + (fxyx · x) + (fxyy · y) \\ fyy = fyyo + (fyyx · x) + (fyyy · y)\end{split}\]

The keywords gradient-x and gradient-y must be given in the same command as the stress keyword. (Use … for line continuation if input is too long.)

stress-zz f

Initialize the \(zz\)-component of stress. The initial out-of-plane stresses in all deformable blocks are set to zero. These stresses can be prescribed with this keyword.

gradient-z fzzx fzzy

where

\[fzz = fzzo + (fzzx · x) + (fzzy · y)\]

gradient-z must be given in the same command as the szz keyword.

water-table fyw

Domains with a \(y\)-coordinate below \(y_w\) are assigned pressure, \(p\), in which

\[p = −ρwgy(yw − y)\]
table i

Domains below the surface defined by table :int:’i’ are assigned pore pressure according to fluid unit weight and the depth below the surface. The fluid density, \(ρ\) (see the block domain initialize density keyword above), and gravity (block mechanical gravity) must be specified prior to the use of this command. if the \(zone-Pressure\) keyword has also been used, pore pressures will also be assigned to the zones.

geometry s

Domains below the surface defined by the geometry :str:’s‘ are assigned pore pressure according to fluid unit weight and the depth below the surface. The fluid density, \(ρ\) (see the block domain initialize density keyword above), and gravity (block mechanical gravity) must be specified prior to the use of this command. if the \(zone-Pressure\) keyword has also been used, pore pressures will also be assigned to the zones.

zone-pressure

If this keyword is included pore-pressures will also be added to the zones. If pore-pressures are specified in the same command as the stresses, this will be interpreted as an initialization of pressure. If the pore-pressure are specified in a separate command (no stress keyword), it will be interpreted as a pore-pressure change and the zone stresses will be adjusted to simulate consolidation or expansion. Note this effect is dependant on the value of Biot’s constant (see block zone property command).

stress-topography

Enables the setting of in-situ stresses based on the density of overburden. This command is used in models where the top surface is irregular, giving rise to significant variation in stress due to overburden. Gravity and material densities must be defined prior to use of this command. The following keyword is available:

kox f

Horizontal stress ratio in the x-direction.

koy f

Horizontal stress ratio (if XUP has been selected).

koz f

Horizontal stress ratio in the z-direction.

xup b

Secifies that x is the vertical direction.

yup b

Secifies that y is the vertical direction (default is y).

overburden f
Set vertical overburden stress to be added to topographic stresses. This is useful if your model does not extend to the actual ground surface. Remember that compressive stresses are negative.
Overburden stresses in the horizontal directions are calculated from the specified ratios.
overburden-xx f

Set xx component of overburden stress. Remember that compressive stresses are negative.

overburden-yy f

Set yy component of overburden stress. Remember that compressive stresses are negative.

overburden-zz f

Set zz component of overburden stress. Remember that compressive stresses are negative.

Notes

  1. UDEC assumes block zone stresses to be total stresses, and joint stresses to be effective stresses. Therefore, the initialization of domain pressures and block/zone stresses must be done by a single INSITU command. Stresses specified by the stress keyword are assumed to be total stresses. Joint stresses are calculated by adding the specified domain pressures to these stresses.
  2. If only stress keywords are given, only zone and contact stresses are set; pore pressures are not changed.
  3. If only pore pressure keywords are given, then only domain pore-pressure conditions are changed, without changing contact/zone effective stresses. If the zone-pressure keyword is given, then zone pore pressures are set. Zone stresses are added to the zone pore-pressure increment, so effective stresses are unchanged. Immediately after the command, the system will not be balanced (e.g., if the water table rises, domain pore pressure will increase, and joints will have to open to accommodate the change). Zones also have to expand as a result of the pore pressure increase.
  4. If stress and pore pressure keywords are given, then stresses are total stresses. Domain pressures, zone total stresses and contact effective stresses are set. If the zone-pressure keyword is present, then zone pore pressures are also set.
  5. Contact stiffnesses and constitutive models must be specified prior to the BLOCK INSITU command in order to calculate joint normal displacement, un, due to in-situ stresses.
  6. For the continuously-yielding joint model, joint initial closure is only calculated for the BLOCK INSITU command if joint normal stiffness is constant (i.e., jen = 0).
