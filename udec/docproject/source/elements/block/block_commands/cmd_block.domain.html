block domain command

Syntax

block domain keyword <range>

Primary keywords:

fix    fix-nonwetting    free    free-nonwetting    group    history    initialize    list    material    property    tolerance    update    well

This command allows fluid pressures to be controled within domains (i.e., “open” spaces between blocks) within the optional range. If the range is omitted, the command applies to all domains except the outer domain. To avoid forming or deleting contacts, use the block contact freeze-new command. When domain pressures are changed, there may be an imbalance between block (total) stresses and joint (effective) stresses. Some cycling may be necessary to bring the model to equilibrium. This command does not change pressures in zones, use the block insitu command to do that.

Total wetting fluid pressure, (\(ppw\)), in a domain is determined by:

\[ppw = pw + dwx · x + dwy · y\]

Total nonwetting fluid pressure, \(ppnw\), in a domain is determined by:

\[ppnw = pnw + dnwx · x + dnwy · y\]
fix

Prevent the wetting fluid pressure from changing.

fix-nonwetting

Specify fixed fluid pressure in nonwetting fluid.

free

Specify that wetting fluid pressure is not controlled in the domains within the range. If the range is omitted, the command applies to all domains except the outer domain. Two things will cause domain pressures to change:

  1. fluid flow; or
  2. domain volume changes (if block fluid flow off).

In both cases, the fluid bulk modulus must be nonzero.

free-nonwetting

Specify that nonwetting fluid pressure is not controlled in the domains within the range. If the range is omitted, the command applies to all domains except the outer domain. (See also block domain free above.)

Two things will cause domain nonwetting pressures to change:

  1. fluid flow; or
  2. domain volume changes (if block fluid flow off).

In both cases, the fluid bulk modulus must be nonzero.

group s

Add domains to a specified group.

history keyword

A time history of domain quantities is stored at regular intervals. The interval default is 10 and can be modified using the history interval command. The following keywords for domain quanitities are available.

fluid-pressure

Domain wetting fluid pressure.

nonwetting-pressure

Domain nonwetting fluid pressure. This will only be active if two phase fluid flow is used.

saturation

Domain fluid saturation. This will only be active if two phase fluid flow is used. If two phase flow is not used, all domains are assumed to be fully saturated. This is true even if the fluid pressure is zero.

initialize keyword <add><mul> :flt:'f'

Set initial conditions within domains. If no range is supplied, the command applies to all domains except the outer domain.

density f

Set the fluid density (\(ρ\)) to be used with block domain initialize table.

fluid-temperature f

Set domain fluid temperature.

gradient-x f

Specify wetting fluid pressure gradient (\(dwx\)) in the \(x\)-direction.

gradient-y f

Specify wetting fluid pressure gradient (\(dwy\)) in the \(y\)-direction.

history keyword

Apply a time history multiplier using one of the following keywords.

cosine ffreq ftime

Get a history of cosine wave load with frequency ffreq (cycles/sec) applied for a problem time period of ftime.

funcname

Use the user-defined FISH function named funcname as the history multiplier.

sine ffreq ftime

Get a history of cosine wave load with frequency ffreq (cycles/sec) applied for a problem time period of ftime.

table i

Input the history via the trick table commands. Table i consists of a list of pairs: \(time\), \(f(time)\). Linear interpolation is performed between the given discrete points.

nonwetting-free

Specify that nonwetting fluid pressure is not controlled in the domains within the range. If the range is omitted, the command applies to all domains except the outer domain. (See also block domain initialize free above.)

nonwetting-gradient-x

Specify nonwetting fluid pressure gradient (\(dnwx\)) in the \(x\)-direction.

nonwetting-gradient-y

Specify nonwetting fluid pressure gradient (\(dnwy\)) in the \(y\)-direction.

nonwetting-pressure f

Specify nonwetting fluid pressure (\(pnw\)).

pore-pressure f

Specify wetting fluid pressure (\(pw\)).

saturation f

Set the domain saturation for two phase fluid flow (SET flow compressible and SET flow incompressible). replace preceding commands The default is 1.0 (saturated).

seepage f

The seepage condition is prescribed on the boundaries of the domain. If a seepage condition is assigned to a domain, then (1) wetting and nonwetting fluid pressures are fixed (wetting fluid pressure is equal to nonwetting fluid pressure); and (2) wetting fluid can inflow into the domain only from saturated domains.

table i

Specify that table i defines the surface of the water table. The command provides fluid pressures in domains corresponding to a static head below the water table surface. The fluid density, \(ρ\) (see the block domain initialize density keyword above), and gravity (block mechanical gravity) must be specified prior to the use of this command. This command does not initialize fluid pressures in zones, use the block insitu command to set pressures in zones.

geometry s

Domains below the surface defined by the geometry :str:’s‘ are assigned pore pressure in domains according to fluid unit weight and the depth below the surface. The fluid density, \(ρ\) (see the block domain initialize density keyword above), and gravity (block mechanical gravity) must be specified prior to the use of this command. This command does not initialize fluid pressures in zones, use the block insitu command to set pressures in zones.

list

List information about wetting or nonwetting fluid in the domains.

nonwetting

List nonwetting fluid information, with the following column headings.

(1) domain index
(2) domain pressure code:
  0   pressure not controlled
  1   pressure controlled
(3) pressure history type
(4) domain pressure
(5) domain area (volume)
(6) flow into (positive) or out of (negative) domain
(7) fictitious domain displacement or, if fluid flow is active, domain area (double precision)
(8) x-coordinate of domain center
(9) y-coordinate of domain center
wetting

List wetting fluid information, with the following column headings.

(1) domain index
(2) domain pressure code:
  0   pressure not controlled
  1   pressure controlled
  2   seepage condition
(3) pressure history type
(4) domain pressure
(5) domain area (volume)
(6) flow into (positive) or out of (negative) domain
(7) fictitious domain displacement or, if fluid flow is active, domain area (double precision)
(8) x-coordinate of domain center
(9) y-coordinate of domain center
material i

Material property number i is assigned to designated domains. (All domains initially default to material = i. The maximum value for i is 50.)

property material i keyword

Capillary pressure parameters in the domain. See Section 2 in the Special Features volume of the UDEC Manual for a detailed explanation of the capillary pressure curve and its use in UDEC.

capillary-alpha f

Set capillary pressure reference hydraulic aperture [length] for material i.

capillary-beta f

Set the \(β\) coefficient from the definition of the capillary pressure curve.

capillary-gamma f

Set the \(γ\) coefficient from the definition of the capillary pressure curve [length]

tolerance keyword
minimum-volume f

Set the minimum domain volume used in the plotting of interior boundaries. (Has no effect on calculation.) PLOT boundary will plot domains that are 50 times this value.

update i

Set the frequency of domain check. The default is every three cycles.

well keyword

Configure a fluid source within a domain.

atdomain fx fy

Locate the fluid source in the domain at position fx, fy.

delete f f

Delete the fluid source in the domain at position fx, fy.

domain i

Locate the fluid source in the domain with index i.

flow f

Set the flow rate of the fluid source. If this value is negative, a sink is assumed.

temperature

Specify temperature of fluid at well.
