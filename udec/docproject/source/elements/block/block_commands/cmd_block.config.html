block config command

Syntax

block config keyword <range>

Primary keywords:

array    axisymmetry    barton-bandis    cellspace    cpp    creep    energy    fluid    plane-stress    thermal

Specify optional calculation modes. These require specifying in advance to allow extra memory to be assigned to each zone or gridpoint. The options are cell space detection logic, plane stress, fluid flow, creep, user-defined models and heat transfer. If any of these options are desired, CONFIG must be given before the BLOCK command. The following keywords apply.

array n

Set the amount of memory (in MB) available in UDEC. This value will be stored in the registry, and will be used for all future UDEC instances. (UDEC must be restarted for this to take effect.)

axisymmetry

Turn on axisymmetry mode. This mode works for mechanical calculation for deformable blocks only. It cannot be used with fluid flow or thermal calculation. Neither is it valid for any structural elements.

barton-bandis

Turn on optional Barton-Bandis joint constitutive model.

cellspace n1 n2

Set alternate contact detection logic. n1 and n2 refer to the number of cells in the \(x\)- and \(y\)-directions. This refers to the method used in UDEC to determine which blocks to consider for contact. This method is slower than the “domain” logic, but allows models with blocks that may bounce from one block to another. Fluid flow is not available with the cell logic.

cpp

Turn on extended and user-defined constitutive models.

creep

Turn on creep analysis.

energy

Turn on energy summation.

fluid

Allocate additional memory structures used for fluid-flow logic in UDEC.

plane-stress

plane stress (only for elastic analysis or zone model strain-softening).

thermal

Turn on thermal analysis.
