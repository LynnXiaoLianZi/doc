block thermal cycle command

Syntax

block thermal cycle keyword

Primary keywords:

age    timestep    implicit    age-off    step    temperature-change    tolerance-gridpoint

Execute thermal timesteps. (See Section 3 in Special Features for details.) Calculation is performed until some limiting condition is reached. The limiting conditions may be the temperature increase at any point, the number of steps, or the simulated age. The limits are changed by the optional keywords listed below. Once a particular limit is specified, it is used for future BLOCK THERMAL CYCLE commands.

age f

Set thermal “heating time” limit [time].

timestep f

The thermal timestep, f, is calculated automatically by the program. This parameter allows the user to change the timestep. If the program determines that this value is too large when the explicit scheme is used, it will automatically reduce the timestep to a suitable value when it begins the analysis. The value determined by the program is usually one-half the critical value for numerical stability. If the program selects a value that causes instability, this option can be used to further reduce the timestep [time].

implicit

Use the implicit scheme instead of the default explicit scheme.

age-off

Turns off the previously requested test for exceeding age f. The default for the age parameter is that the age is not tested until an age has been explicitly requested via an “age = value” following a BLOCK THERMAL CYCLE command.

step i

Set thermal step limit (default: i = 100,000).

temperature-change f

Set maximum total temperature change, f, since the previous mechanical cycles (default: f = 20) [degrees].

tolerance-gridpoint f

Gridpoints within tolerance, f, from each other are combined for thermal calculations (default: f = 0.1) [length].

Old limits apply upon restart. When a BLOCK THERMAL CYCLE command has been completed, the program will indicate which parameter has caused it to terminate. To ensure that the run stops for the correct limit, the values of the others should be set very high.

The explicit scheme is always used unless the keyword implicit follows the BLOCK THERMAL CYCLE command.
