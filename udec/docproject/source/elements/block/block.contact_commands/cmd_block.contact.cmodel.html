block contact cmodel command

Syntax

block contact cmodel keyword <range>

Primary keywords:

assign    default    default    list    load

There are presently two independant methods for assigning constitutive models and properties to contacts[1]. The method described here assigns a constitutive model to each contact and that contact has its own local set of properties. This method uses more memory, but is more flexible. Using this method, FISH may be used to modify the properties during cycling. The basic form of this command will associate a constitutive model with contacts in the specified range. This command can also be used to assign properties to the contacts by following the model name with the property keywords. Contact model properties may also be assigned using the block contact property command. At present, five joint constitutive models are available through the use of this command: Coulomb slip (point contact); Coulomb slip (area contact); Coulomb slip with residual strength (area contact); continuously yielding; and Barton-Bandis (optional model).

This command is also used to load and assign user defined contact (joint) models. The user defined contact models is an optional feature in UDEC. At present, one example user defined joint constitutive model is available through use of this command. This model is described in Section 4.2.2 in Constitutive Models.

The model must be assigned before assigning properties. If properties that are not consistent with the chosen model are given, a warning message will be issued to inform the user that the unneeded properties were not accepted. If a required property is not specified, the default will be used. WARNING: When a model is invoked over a specified range, all properties associated with that model are initialized to zero in that range. Properties previously assigned to that range must be specified again, even if their values have not changed.

Examples of this command are:

block contact cmodel assign area

block contact cmodel assign area stiffness-normal 2e9 stiffness-shear 1e9

The model and properties that are assigned to new contacts that are formed during cycling can also be specified using this command. An example of this usage is:

block contact cmodel default area stiffness-normal 2e9 stiffness-shear 1e9

block comtact cmodel default 2 material 1

The Coulomb slip models are described in Section 1.2.4 in Theory and Background, the continuously yielding model is described in Section 2 in Constitutive Models, and the Barton-Bandis model is described in Section 3 in Constitutive Models. Any joint constitutive model that already exists for some or all of the contacts in the given range (as specified by the block contact change model command) is ignored. The block contact default-model command may be used to specify a model and properties to be used for new contacts.

assign

Assign constitutive models via the following names.

area

Assign the Coulomb slip (area) contact model. This model differs from the point model by having a length (and width of 1). This allows stress to be calculated. The property keywords for the area model may be found here.

barton-bandis

Assign the Barton-Bandis contact model (optional model). The property keywords for the Barton-Bandis model may be found here.

continuously-yielding

Assign the continuously yielding contact model. The property keywords for the c-y model may be found here.

point

Assign the Coulomb slip (point contact) model.

Assign the elastic contact model. The property keywords for the elastic model may be found here.

residual

Assign the Coulomb slip with residual strength model. This model difers from the Coulomb model by having post peak (residual) properties. The property keywords for the residual model may be found here.

default imodel material imat

Set the default material number for new contacts (created after cycle 1). If material is not specified, it is assumed equal to the block material number on one side of the joint.

default smodel ... :str:'sprop' fvalue

Set the default constitutive relation for new contacts (created after cycle 1). If default is not specified, it is assumed equal to 2. Thhe property keywords and values must also be included.

list
names <s >

This will list all of the models that are availble to be assigned using this command. This wil include any user defined models that have been loaded (if any).

properties <s >

This will list all of the models and the poperty keyword names that are availble to be assigned using this command. This wil include any user defined models that have been loaded (if any).

states <s >

This will list the names of the model states associated associated with the available contact (joint) models.

load s

Load a user-defined joint constitutive model (UDM). The UDM option is required to use this command. Also, CONFIG cppudm must be used for the command to be accepted. Models exist as dynamic linked libraries, and must be loaded prior to assignment. Models that are placed in the “pluginsjmodels” folder are be automatically loaded at runtime. Models loaded with the load keyword are designated by their file name (s).

Table 1: Property keywords for Area Coulomb slip constitutive model
  keyword Description
(1) aperture-residual residual aperture at high normal stress [length]
(2) aperture-zero-load aperture for zero normal stress [length]
(3) cohesion joint cohesion [stress]
(4) constant-b emperical multiplier for fluid flow cubic law (default = 1)
(5) dilation-limit shear displacement at which joint stops dilating. (default=0) (no limit if = 0) [length]
(6) exponent exponent for fluid flow cubic law (default = 3)
(7) friction joint friction [degrees]
(8) friction-residual residual joint friction [degrees]. The residual joint friction (if specified)cannot be exactly equal to zero (use a small nonzero value)
(8) nonwetting-permeabilty-factor non-wetting fluid joint permeability constant (see eq. 2.3 in Special Features Manual). also called joint permeability factor [1/(stress*time)]
(9) permeability-factor wetting fluid joint permeability constant (see eq. 2.3 in Special Features Manual). Also called joint permeability factor [1/(stress*time)]
(10) stiffness-normal joint normal stiffness [stress/length]
(11) stiffness-normal-table joint normal stiffness table
(12) stiffness-shear joint shear stiffness [stress/length]
(13) tension joint tensile strength [stress]

Note

The default value for these properties (with the exception, as noted, of constant-b, exponent and friction-residual) is zero. Joint model 5 is similar to joint model 2, except that when the joint tensile or shear strength is exceeded, then joint tensile strength, joint friction angle and joint cohesion are set to residual values. For joint model 2, joint tensile strength, joint friction angle and joint cohesion are preserved regardless of loading path. To limit fluid flow to broken joint segments use the command block contact crackflow on.

Table 2: Property keywords for Barton Bandis constitutive model
  keyword Description
(1) aperture initial aperture for Barton Bandis joint. This value will be calculated from an emperical formula if not specified [length]
(2) bbdmax maximum value of dilation angle [degrees]
(3) bbdmin minimum value of dilation angle [degrees]
(4) bbdplateau determines whether the damage factors are applied to the plateau region of the shear stress values during shear reversal. Setting this parameter equal to 1 will result in more realistic dilation calculations. The default (bbdplateau = 0) is consistent with the original definition of the Barton-Bandis model.
(5) bbdtable Barton-Bandis shear damage factor table. Up to 10 damage factors may be specified in a table by using the TABLE command. The factors are used to reduce the peak shear for each shear cycle. The first value is normally 1.0 and is applied to shear cycle 1. Each additional value will apply to the next shear cycle. The last value given will be used for all subsequent cycles. The default factors are: 1.0; 0.75; 0.625; 0.5625; and 0.53125
(6) jcso laboratory scale joint wall compressive strength [stress]
(7) jkn joint normal stiffness [stress/length]
(8) jks joint shear stiffness [stress/length]
(9) jrco laboratory scale joint roughness coefficient
(10) ln field scale join length [length]
(11) lo laboratory scale joint length [length]
(12) phir residual friction angle [degrees]
(13) sigmac intact rock uniaxial compressive strength [stress]

Note

The default value for these properties (with the exceptions, as noted) is zero.

Note

The default values for exponent-normal, exponent-shear, friction-initial and friction-residual are zero. If neither stiffness-normal-minimum nor stiffness-normal-maximum is defined, joint normal stiffness will be constant (i.e., exponent-normal will have no effect). Joint shear stiffness is defined in a similar manner. Timestep calculations for cy joints are based on the values of stiffness-normal-maximum and stiffness-shear-maximum. In general, it will not be necessary to further reduce the timestep for numerical stability. The user should avoid very large values for stiffness-normal-maximum or stiffness-shear-maximum, as these might result in very small timesteps.
