block edge apply-interior command

Syntax

block edge apply-interior keyword <range>

Primary keywords:

stress    gradient-x    gradient-y    domain    in-excavation    history    delete

Stresses are applied to block edges of an interior boundary (i.e., not lying on the outer boundary) within a specified range. An alternative is to use the domain or :tok`in-excavation` keywords to specify the void over which the stresses will be applied.

stress fxxo fxyo fyyo

Apply boundary stress parameters: \(xx\)-stress, \(xy\)-stress and \(yy\)-stress.

gradient-x fxxx fxyx fyyx

Linearly vary boundary stress in X direction. :math: \(sxxx\), \(sxyx\), \(syyx\), assuming \(sxxo\), \(sxyo\) and \(syyo\) are stresses at origin (0,0).

gradient-y fxxy fxyy fyyy

Linearly vary boundary stress in Y direction. \(sxxy1, `sxyy\), \(syyy\), assuming \(sxxo\), \(sxyo\) and \(syyo\) are stresses at origin (0,0).

Linearly varying boundary stress equation.

\[\begin{split}sxx = sxxo + (sxxx \cdot x) + (sxxy \cdot y) \\ sxy = sxyo + (sxyx \cdot x) + (sxyy \cdot y) \\ syy = syyo + (syyx \cdot x) + (syyy \cdot y) \\\end{split}\]
domain i

The keyword domain i specifies the interior domain over which the stress are applied. The domain may be determined by turning on the index in a domain plot or by using the block domain list. Note that the id of a domain will change if the model is changed. It is better to use the in-excavation keyword. This keyword does not work in conjunction with CONFIG CELL.

in-excavation fxx fyy

The keyword in-excavation is used to specify a point inside of the void that the stresses will be applied to. This keyword does not work in conjunction with CONFIG CELL.

history

The keyword history defines a time history that is a multiplier for all the stresses applied.

Boundary loads, stresses or velocities can be applied as a constant, sinusoidal or user-defined FISH function, or they may consist of a series of points given by a TABLE command or read from a file. The keywords must be given on the same line specifying the load, stress or velocity.

cosine ffreq ftime

Get a history of cosine wave load with frequency ffreq (cycles/sec) applied for a problem time period of ftime.

funcname

Use the user-defined FISH function named funcname as the history multiplier.

sine ffreq ftime

Get a history of cosine wave load with frequency ffreq (cycles/sec) applied for a problem time period of ftime.

table i

Input the history via the trick table commands. Table i consists of a list of pairs: \(time\), \(f(time)\). Linear interpolation is performed between the given discrete points.

delete

Delete interior boundaries should be deleted prior to deleting blocks that are part of the boundary.
