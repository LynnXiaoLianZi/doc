block edge apply command

Syntax

block edge apply keyword <range>

Primary keywords:

convection    dynamic-free-field    flux    free-x    free-y    history    history-time-reference    history-normal    history-shear    history-x    history-y    impermeable    interior    multiply-normal    multiply-shear    multiply-x    multiply-y    nonwetting-gradient-x    nonwetting-gradient-y    nonwetting-impermeable    permeable    nonwetting-permeable    nonwetting-pressure    pore-pressure    pressure-gradient-x    pressure-gradient-y    radiation    saturation    saturation-gradient-x    saturation-gradient-y    seepage    stress-principal    stress

Boundary conditions are specified for edges of blocks. This functionality is limited to the edges that lie on the outer domain (external boundary) of a block model. If block config cell has been used, the boundary conditions will be applied to any block edges within the specified range. If the range is omitted, the boundary condition applies to the entire boundary. Boundary stresses can be applied to both rigid and deformable blocks (use block gridpoint apply to set velocity boundary conditions). Boundary conditions are defined by a given boundary condition keyword as follows.

convection f1 f2

Apply a thermal boundary condition. f1 is the convective heat transfer coefficient (w/m2 ◦C). f2 is the temperature of the medium at which convection occurs. To remove a convection boundary condition, apply the same heat transfer coefficient with a negative sign. It is not physically realistic to use a negative heat transfer coefficient in any other circumstances. Unless otherwise specified, all boundaries are assumed to be insulated.

dynamic-free-field

Apply a free-field boundary condition automatically to the lateral outer boundaries of the model. The dynamic free-field consists of a one-dimensional finite-difference calculation that executes in parallel with the main stress-strain calculations, and provides the lateral boundary conditions for a dynamic analysis in which a vertical propagating plane wave is applied to the base of the model. The boundaries must be vertical and straight. The block edge apply dynamic-free-field command must be given before changing other boundary conditions for the dynamic stage of an analysis. The properties and constitutive models of the zones adjacent to the free-field boundary are copied and used by the dynamic free-field. The model should be in static equilibrium prior to using the command. The free-field itself is automatically cycled to equilibrium when the command is given. The viscous boundary conditions along the sides of the model are applied automatically. The free-field performs a small-strain calculation even though the adjacent zones may be in large-strain. The block dynamic free-field command may be used instead of, or in addition to, the block edge apply free-field command to manually control the free-field, but it is usually not needed. See Section 4.3.1.4 in Special Features for more details. The free-field cannot be used with block config cell.

flux f1 f2
Apply a thermal flux boundary condition. f1 is the initial flux (watts/m:sup:\(2\)). f2 is the decay constant (s−1). To remove a flux condition, a new condition should be applied with the strength replaced by Srep where:
Srep = −Sini × exp[cd (tcurr − tini )]
free-x

Remove a boundary condition and all loads in the \(x\)-direction.

free-y

Remove a boundary condition and all loads in the \(y\)-direction.

history

The keyword history defines a time history that is a multiplier for all the stress, and hydraulic pressures (e.g., pore-pressure, pressure-gradient-x and pressure-gradient-y) given on the same input line.

Boundary stresses can be applied as a constant, sinusoidal or user-defined FISH function, or they may consist of a series of points given by a TABLE command or read from a file. The keywords must be given on the same line specifying the load, stress or velocity.

cosine ffreq ftime

Get a history of cosine wave load with frequency ffreq (cycles/sec) applied for a problem time period of ftime.

funcname

Use the user-defined FISH function named funcname as the history multiplier.

sine ffreq ftime

Get a history of cosine wave load with frequency ffreq (cycles/sec) applied for a problem time period of ftime.

table i

Input the history via the trick table commands. Table i consists of a list of pairs: \(time\), \(f(time)\). Linear interpolation is performed between the given discrete points.

history-time-reference keyword

Set the reference time used in stress and pressure histories for the BLOCK EDGE APPLY command (default = mechanical).

flow

Use fluid flow time for histories.

mechanical

Use mechanical time for histories.

history-normal

Set a history in the normal direction.

history-shear

GSt a history in the shear direction.

history-x

Set a history in the \(x\)-direction.

history-y

Set a history in the \(y\)-direction.

impermeable

Apply an impermeable boundary for wetting fluid.

interior keyword

Stresses are applied to block edges of an interior boundary (i.e., not lying on the outer boundary) within a specified range. For example, a range domain i restricts the range to block edges on the boundary of interior domain i.

stress fxxo fxyo fyyo

Apply boundary constant stress parameters: (\(sxxo\), \(sxyo\), and \(syyo\)).

gradient-x fxxx fxyx fyyx

Linearly varying interior boundary stress in X direction. (\(sxxx\), \(sxyx\), \(syyx\)), assuming \(sxxo\), \(sxyo\) and \(syyo\) are stresses at origin (0,0).

gradient-y fxxy fxyy fyyy

Linearly varying interior boundary stress in Y direction. (\(sxxy1\), \(sxyy\), \(syyy\)), assuming \(sxxo\), \(sxyo\) and \(syyo\) are stresses at origin (0,0).

Linearly varying boundary stress equation.

\[\begin{split}sxx = sxxo + (sxxx \cdot x) + (sxxy \cdot y) \\ sxy = sxyo + (sxyx \cdot x) + (sxyy \cdot y) \\ syy = syyo + (syyx \cdot x) + (syyy \cdot y) \\\end{split}\]
history

The keyword history defines a time history that is a multiplier for all the interior boundary stress.

Boundary loads, stresses or velocities can be applied as a constant, sinusoidal or user-defined FISH function, or they may consist of a series of points given by a TABLE command or read from a file. The keywords must be given on the same line specifying the load, stress or velocity.

cosine ffreq ftime

Get a history of cosine wave load with frequency ffreq (cycles/sec) applied for a problem time period of ftime.

funcname

Use the user-defined FISH function named funcname as the history multiplier.

sine ffreq ftime

Get a history of cosine wave load with frequency ffreq (cycles/sec) applied for a problem time period of ftime.

table i

Input the history via the trick table commands. Table i consists of a list of pairs: \(time\), \(f(time)\). Linear interpolation is performed between the given discrete points.

delete

Delete interior boundaries should be deleted prior to deleting blocks that are part of the boundary.

multiply-normal f

Normal-direction load multiplier (default = 1.0)

multiply-shear f

Shear-direction load multiplier (default = 1.0)

multiply-x f

\(x\)-direction load multiplier (default = 1.0)

multiply-y f

\(y\)-direction load multiplier (default = 1.0)

nonwetting-gradient-x f

Apply non-wetting fluid pressure gradient along the \(x\)-axis.

nonwetting-gradient-y f

Apply non-wetting fluid pressure gradient along the \(y\)-axis.

nonwetting-impermeable

Apply an impermeable boundary for non-wetting fluid.

permeable

Apply a permeable boundary to fluid for joints intersecting the boundary.

nonwetting-permeable

Apply a permeable boundary to non-wetting fluid for joints intersecting the boundary.

nonwetting-pressure f

Apply a non-wetting fluid pressure at boundary (the default is f = 0.0)

pore-pressure f

Apply wetting fluid pressure at boundary; must still use the stress keyword to set mechanical boundary conditions. (The default is f = 0.0.)

pressure-gradient-x f

Apply wetting fluid pressure gradient along the \(x\)-axis; must still use the xload, yload or stress keyword to set mechanical boundary conditions.

pressure-gradient-y f

Apply wetting fluid pressure gradient along the \(y\)-axis; must still use the xload, yload or stress keyword to set mechanical boundary conditions.

radiation f1 f2

Apply a thermal boundary condition. f1 is the radiative heat transfer coefficient (for black bodies, this is the Stefan-Boltzmann constant, 5.668 × 10−8 w/m2 K4). f2 is the temperature of the medium to which radiation occurs. To remove a radiation boundary condition, apply the same heat tansfer coefficient with a negative sign. It is not physically realistic to use a negative heat transfer coefficient in any other circumstances. Unless otherwise specified, all boundaries are assumed to be insulated.

saturation

Apply wetting fluid saturation on the boundary

saturation-gradient-x

Apply saturation gradient along the \(x\)-axis

saturation-gradient-y

Apply saturation gradient along the \(y\)-axis

seepage

Apply a seepage boundary condition. If a seepage boundary condition is assigned to a boundary, then (1) wetting and non-wetting fluid pressures are fixed (wetting fluid pressure is equal to non-wetting fluid pressure); and (2) the unsaturated boundary is impermeable for wetting fluid.

stress-principal keyword

Alternate method to define stress at a boundary by specifying a stress measurement at a point and an optional gradient. The use of this command requires that the global orientation of the UDEC plane be specified using the commands model orientation dip-direction and model orientation origin. The keywords are:

s1 f

major principal stress

s2 f

intermediate principal stress

s3 f

minor principal stress

dip-s1 f

major principal stress dip

dip-direction-s1 f

major principal stress direction

dip-s2 f

dip intermediate principal stress dip

dip-direction-s2 f

intermediate principal stress direction

The following are optional keywords:

gradient-vertical f

gradient in the veritcal direction

location fx fy fz

The location specifies the coordinates of the stress measurement. The gradient is the stress change in the vertical direction. All other stresses are incremented proportionately by the gradient to prevent stress rotation.

stress fxxo fxyo fyyo <keyword>

Apply boundary constant stress parameters: (\(sxxo\), \(sxyo\), and \(syyo\)).

gradient-x fxxx fxyx fyyx

Linearly vary boundary stress in X direction. (\(sxxx\), \(sxyx\), \(syyx\)), assuming \(sxxo\), \(sxyo\) and \(syyo\) are stresses at origin (0,0).

gradient-y fxxy fxyy fyyy

Linearly vary boundary stress in Y direction. (\(sxxy\), \(sxyy\), \(syyy\)), assuming \(sxxo\), \(sxyo\) and \(syyo\) are stresses at origin (0,0).

Linearly varying boundary stress equation.

\[\begin{split}sxx = sxxo + (sxxx \cdot x) + (sxxy \cdot y) \\ sxy = sxyo + (sxyx \cdot x) + (sxyy \cdot y) \\ syy = syyo + (syyx \cdot x) + (syyy \cdot y) \\\end{split}\]

Note

All stresses are assumed to be constant and permanent by default and are added to the existing permanent loads.

Warning

Stress conditions should always be applied prior to any velocity conditions acting on the same location. The stress boundary affects all degrees of freedom. For example, if block edge apply stress follows a :cmd:`block gridpoint apply velocity-x command affecting the same gridpoint(s), then the effect of the previously prescribed velocity condition will be lost.
