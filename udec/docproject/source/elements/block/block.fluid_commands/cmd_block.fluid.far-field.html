block fluid far-field command

Syntax

block fluid far-field keyword

Primary keywords:

list    permeability-xx    permeability-xy    permability-yy    pore-pressure    gradient-x    gradient-y    mesh-radius    mesh-ratio    mesh-elements

Create a porous-medium grid around the outer boundary of the model with this command. The following keywords and associated parameters are available.

list

List data for the porous medium outer boundary, including a listing of fluid pressures and node coordinates. The column headings are:

(1) grid node number
(2) pressure at node
(3) x-coordinate of node
(4) y-coordinate of node
(5) node pressure code:
  0     pressure not fixed
  1     pressure fixed
(6) index of boundary corner linked to node
permeability-xx f

Set the \(xx\)-component of the permeability tensor.

permeability-xy f

Set the \(xy\)-component of the permeability tensor.

permability-yy f

Set the \(yy\)-component of the permeability tensor.

pore-pressure f

Set initial fluid pressure at boundary and throughout porous-medium grid. (If a gradient is specified, f is the pore pressure at the origin of the grid.)

gradient-x f

Specify initial pressure gradient along the \(x\)-axis.

gradient-y f

Specify initial pressure gradient along the \(y\)-axis.

mesh-radius f

Specify total radius of mesh, with origin at the centroid of the block.

mesh-ratio f

Specify the ratio between radial size of an element on adjacent element. (The default f = 1.)

mesh-elements i

Set the number of radial elements around the outer boundary. (The default is i = 2.)

Notes

  1. Nodes at the outer boundary of the grid retain a fixed fluid pressure.
  2. The fluid boundary must be created for a single-block system (i.e., before any joints are created). The number of fluid-boundary zones in a circumferential direction is equal to the number of sides between corners on the single block that is created. Use the block create corner-angle command to control the number of corners.
  3. A block edge apply or a block gridpoint apply command must be given before cycling, because this boundary serves to transfer information between the flow calculations in the rock joints and the flow calculation in the porous-medium region. For example, the command BLOCK EDGE pore-pressue = 0 will establish the boundary linked list.
  4. At present, the porous-medium logic is only intended for steady-state flow analysis; a relaxation factor is used for convergence.
  5. The formulation is small-strain only; it does not allow for changes in boundary linkages.
  6. The formulation is for fully saturated flow. No free-flow surface may be modeled.
  7. See Section 2.2.7 in Special Features for an explanation of the formulation, and example applications.
